1. Our Core Principles & Features

This application is built on several core principles, each supported by specific features. Understanding these will help you see the full power of the tool.

Uncompromising Security

Your data is sensitive, so security is our top priority. Access is strictly controlled through a secret Invitation Code for registration. All user passwords are encrypted using the `bcrypt` algorithm, and every action requires a valid login token (JWT). This multi-layered approach ensures your financial and donor information remains confidential.

Absolute Financial Accountability

For an NGO, trust is everything. The Expense Tracker is more than just a ledger; it's an accountability engine. It features an immutable audit log (`ExpenseLog`) that records every single creation, update, or deletion of a financial entry. This means you have a permanent, unchangeable history of all financial activities, which is critical for audits and building donor confidence.

Centralized & Clear Governance

Important decisions can't get lost in conversation. The Meeting Management feature, restricted to Super Admins, serves as an official record-keeper. By documenting the agenda, topics discussed, and conclusions, it creates a single source of truth for your organization's governance, ensuring everyone is aligned and decisions are tracked.

Effortless Donor Relations

Building relationships with supporters is key. The application streamlines this by integrating a direct acknowledgement system. With a single click, you can send a personalized "Thank You" message via WhatsApp or SMS using Twilio integration. This simple action helps make your donors feel valued and keeps them engaged with your cause.

4. Managing Donations

Why is this a game-changer?

This isn't just a list; it's a complete relationship and transparency tool. By logging every donation, you build trust with your donors, create flawless financial records for audits, and get a clear picture of your fundraising success.

Visual Guide: The Donation Form

Log New Donation

Form Fields Explained

  1. Donor's Name: The full name of the person donating.
  2. Collected By: Your name (or another team member's).
  3. Donor's Location: Their city/village.
  4. Donation Type: Choose 'Money' for cash or 'Item' for goods.
  5. Donor's Contact: The phone number for sending acknowledgements.
  6. Amount / Item Description: Enter the value or a description based on the type.

5. Tracking Expenses (Super Admin)

Why is this a game-changer?

This module provides complete financial control and transparency. It allows you to track not only your expenses (debits) but also other forms of income like grants (credits). The immutable audit log (`ExpenseLog`) ensures that every financial entry and deletion is permanently recorded, creating an un-editable history for maximum accountability.

Real-World Scenario

You spend ₹2,000 on fuel for a relief vehicle, logging it as a "Debit." The next day, you receive a ₹10,000 grant from a local business, which you log as a "Credit." The system automatically calculates your new total balance. At the end of the year, your auditor asks for financial records. You provide them with two files: the "Current Expenses" CSV for the final summary and the "Audit Log" CSV to prove that no records were tampered with.

Visual Guide: The Expense & Income Form

Log Expense/Income

Form Fields Explained

  1. Date: Select the date the transaction occurred. You cannot select a future date.
  2. Type: Choose "Debit (Expense)" for money spent or "Credit (Income)" for money received.
  3. Amount (₹): Enter the transaction amount.
  4. Purpose / Details: Be specific. Instead of "Supplies," write "Purchase of stationery for office."

6. Managing Meetings (Super Admin)

Why is this a game-changer?

This feature transforms verbal discussions into official, searchable records. It ensures that decisions made in meetings are documented and not forgotten. It provides a single source of truth for your team about what was discussed, what was decided, and who is responsible for what.

Real-World Scenario

After the monthly review meeting, the Super Admin uses the "Add New Meeting" feature to log the details. They list the members who were present, the topics discussed (e.g., "budget shortfall"), and the conclusion ("ACTION: Rohan to find new funding sources"). Three months later, when someone asks, "Who was supposed to handle the budget?", anyone can look up this record and find the answer instantly.

Visual Guide: The Meeting Log Form

Log New Meeting

Form Fields Explained

  1. Date & Time: Select the exact date and time the meeting took place.
  2. Members Present: Type the names of all attendees, separated by commas.
  3. Agenda: Briefly state the main purpose of the meeting.
  4. Topics Discussed: Provide a more detailed summary of the conversation and key points.
  5. Conclusion: This is the most important part. Document the final decisions or action items.

Data Interfaces

The application uses the following TypeScript interfaces to structure its data on the frontend.

Donation

interface Donation {
  _id: string;
  donorName?: string;
  location: string;
  contact: string;
  donationType?: 'money' | 'item';
  amount?: number;
  itemDescription?: string;
  collectedBy: string;
  loggedBy?: string;
  acknowledged: 'No' | 'WhatsApp' | 'SMS';
}

Expense

interface Expense {
  _id: string;
  date: string;
  type: 'credit' | 'debit';
  amount: number;
  purpose: string;
  loggedBy: string;
}

Meeting

interface Meeting {
  _id: string;
  date: string;
  membersPresent: string[];
  agenda: string;
  topicsDiscussed: string;
  conclusion: string;
  writtenBy: string;
}

User

interface User {
  _id: string;
  name: string;
  username: string;
  isSuperAdmin?: boolean;
}
